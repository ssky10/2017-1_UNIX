프로세스 : 현재 실행 중인 프로그램
 - 데몬 프로세스 : 유닉스 커널에 의해서 실행되고 특정 서비스를 제공하기 위해 존재
 - 일반 프로세스
    - 부모 프로세스 : 다른 프로세스를 만듦, 부팅시 실행되는 1번 프로세스를 제외한 모든 프로세스가 가지고 있음
    - 자식 프로세스 : 부모프로세스에 의해 만들어지는 프로세스, 작업 종료시 부모 프로세스에 결과 반환
       => 부모 프로세스가 선 종료시 자식 프로세스=고아 프로세스
       => 부모 프로세스에 종료 신호 전송에도 불구하고 종료X = 좀비 프로세스

프로세스 목록 보기
 - ps [옵션] : 현재 실행중인 프로세스에 대한 정보 출력
      * -e : 시스템에서 실행중인 모든 프로세스 정보 출력
      * -f : 각 프로세스에 대한 자세한 정보 출력
      * -u uid : 특정 사용자에 대한 모든 프로세스 출력
 - pgrep [옵션] [패턴] : 지정한 패턴과 일치하는 프로세스에 대한 정보 출력
         * -x : 패턴과 정확히 일치하는 프로세스 정보 출력
         * -n : 패턴을 포함하고 있는 가장 최근의 프로세스 정보 출력
         * -u uid : 특정 사용자에 대한 모든 프로세스 출력
         * -l : PID와 프로세스 이름 출력
         * -t term : 특정 단말기와 관련된 프로세스 정보 출력

프로세스 종료
 - kill [시그널] PID
         * -15 : 종료(디폴트 값)
         * -9 : 강제종료
 - pkill [시그널] 프로세스명
          * -15 : 종료(디폴트 값)
          * -9 : 강제종료

작업제어
 - jobs [옵션] : 백그라운드 작업 출력
        * %번호 : 해당번호의 작업정보 출력
        * %+(%%) : 작업순서가 +인 작업정보를 출력
        * %- : 작업순서가 -인 작업 정보를 출력
   - 출력형태 : [작업번호] 작업순서 상태 명령
 - bg [%작업번호] : 현재작업이나 특정작업을 백그라운드로 전환
 - fg [%작업번호] : 현재작업이나 특정작업을 포그라운드로 전환
 - ctrl+z : 포그라운드 작업을 중지시키고, 백그라운드의 중지 된 목록으로 보냄
 - nohup 백그라운드명령 : 명령의 실행결과와 오류메시지를 현재 디렉토리에 nohup.out파일에 저장
